## 1. Port Congestion Intelligence Layer

**Category:** human
**Manager:** `portCongestionManager.js`
**Proposed file:** `proposed/port-congestion-intelligence/portCongestionManager.js`

### Strategic Value

Port congestion is a leading indicator for supply-chain disruption, sanctions enforcement effectiveness, and economic coercion. When a port backs up (e.g., Port Said during Red Sea diversions, Rotterdam during energy embargoes), the delay ripples through manufacturing output and military resupply timelines. Overlaid on Vanguard1's vessel tracks and chokepoint glyphs, a congestion layer lets an analyst instantly correlate vessel clustering near anchorages with geopolitical events — and flag anomalous queuing patterns (ships waiting longer than baseline may indicate sanctions inspection, port access denial, or labor action).

Real-world example: In 2023–2024, when Houthi Red Sea attacks rerouted vessels around the Cape of Good Hope, Port Said's congestion score dropped sharply while Cape Town and Singapore surged — a pattern visible days before it appeared in shipping news.

### Data Source

```
Portcast Port Congestion API
GET https://api.portcast.io/v1/port-congestion?api_key=YOUR_KEY

Returns: vessel count at anchorage, median wait time (hrs), congestion score 0–100, UNLOCODE
Free tier: available for research use
Docs: https://portcast.io/blog/portcast-port-congestion-data-now-available-via-api

Alternative: MarineTraffic Terminal Congestion API
GET https://services.marinetraffic.com/api/expectedarrivals/{API_KEY}/portid:1/protocol:jsono
```

### Visual Concept

Pulsing semi-transparent rings overlaid on each major port marker. Ring radius scales with vessels at anchor. Ring colour follows green → amber → red keyed to congestion score (0–100). A soft glow disc behind each ring creates an aura, with intensity proportional to severity. Rings pulse at ~1.2 Hz — a severely congested port visually "throbs" on the map. 15 ports hardcoded with static baseline data as demo fallback.

### Implementation Approach

`THREE.RingGeometry` per port flat on ocean plane (Y = 0.05). `THREE.CircleGeometry` glow disc. `MeshBasicMaterial` with transparency. Per-frame pulse via `Math.sin(phase)` driving opacity and scale. Colour from linear interpolation between SCORE_COLOURS ramp. LOD: hidden above camera.y = 180. API polling every 10 minutes via async `_fetchData()`.

### Research Notes

Portcast, GoComet, and Safecube all provide congestion APIs. Northern European ports saw dramatic congestion increases in 2025 (Antwerp +37%, Hamburg +49%, Bremerhaven +77% from spring to early summer 2025). MarineTraffic Terminal Congestion provides terminal-level granularity — not just port-level. For highest signal density, combine Portcast score with aisManager vessel counts near known anchorage zones.

## 2. Arctic Sea Ice & Polar Route Navigator

**Category:** atmosphere
**Manager:** `arcticIceManager.js`
**Proposed file:** `proposed/arctic-ice-routes/arcticIceManager.js`

### Strategic Value

The Arctic is a rapidly evolving strategic theatre. Russia's Northern Sea Route (NSR) is a declared national priority — in ice-free months it cuts transit time between Europe and East Asia by ~30% vs. the Suez Canal. This layer lets analysts see when the NSR is passable, correlate it with live AIS vessel tracks, flag anomalous vessel movements in ice-marginal zones, and track seasonal change as a climate-as-intelligence signal. 2025 Arctic sea ice extent was at near-record lows, meaning the NSR had its longest open season on record — a geopolitically significant development not visible in any current Vanguard1 layer.

### Data Source

```
NSIDC Sea Ice Index — WMS API (free, no key required)
GET https://nsidc.org/api/mapservices/NSIDC/wms
    ?SERVICE=WMS&VERSION=1.1.1&REQUEST=GetMap
    &LAYERS=G02135_north_concentration_hdf5
    &BBOX=-180,55,180,90
    &WIDTH=1024&HEIGHT=512&SRS=EPSG:4326&FORMAT=image/png
    &TIME=2026-06-09

Returns: PNG where pixel brightness ≈ ice concentration (0–100%)
Updated: daily
Docs: https://nsidc.org/data/user-resources/help-center/guide-nsidc-data-map-services-api

Alternative: JAXA AMSR2 (6.25 km resolution, free with academic registration)
https://gcom-w1.jaxa.jp/auth.html
```

### Visual Concept

A translucent white-blue mesh overlays the polar cap (above ~58°N). Where WMS data is available, the mesh texture directly reflects satellite-measured concentration — dense pack ice appears opaque, open leads nearly invisible. Three polar shipping routes render as coloured lines over the ice: Northern Sea Route, Northwest Passage, Transpolar Route (future). Animated chevron dots flow along open/marginal routes. The ice edge visually indicates route passability at a glance.

### Implementation Approach

`THREE.PlaneGeometry` (64×32 segments) covering polar cap, flat on ocean plane. Vertex colour gradient (blue-white) as fallback. On successful WMS fetch: `THREE.TextureLoader` applies satellite PNG as `material.map` and `material.alphaMap`. `THREE.Line` per route, colour set from status at build time. `THREE.Points` chevron particles per route. Daily refresh interval. No blocking API key requirement — works immediately with procedural ice field.

### Research Notes

NSIDC WMS requires no API key and is freely accessible. Test immediately: open the WMS URL in a browser to see yesterday's ice concentration PNG. Note: NSIDC's Sea Ice Today *web service* reduced hours in Oct 2025 due to funding, but the WMS API for the Sea Ice Index dataset remains operational. JAXA AMSR2 provides 6.25 km resolution vs. NSIDC's 25 km — worth integrating as secondary source for close-zoom views. Cross-referencing vessel tracks from aisManager with ice boundary positions could surface vessels operating in normally-impassable zones.

await arcticIceManager.init(scene);
// In animation loop:
