# Vanguard1: Volumetric Atmosphere & Sky — Three.js r184 / WebGL2 Implementation Guide

## 1. Best Available Three.js / WebGL Implementations

### 🥇 Tier 1: `@takram/three-geospatial` (The Gold Standard)

**GitHub:** https://github.com/takram-design-engineering/three-geospatial
**NPM:** `@takram/three-atmosphere` + `@takram/three-clouds`

This is unambiguously the best option available. 
The project provides modular packages including `@takram/three-atmosphere` (Precomputed Atmospheric Scattering, Beta) and `@takram/three-clouds` (Geospatial volumetric clouds, Beta).



It is a Three.js and R3F implementation of Eric Bruneton's Precomputed Atmospheric Scattering.
 
Latest version is 0.17.1, last published 10 days ago.


**What `@takram/three-atmosphere` provides:**
- 
Precomputed textures: `irradianceTexture`, `scatteringTexture` (Data3DTexture), `transmittanceTexture`, `singleMieScatteringTexture` (Data3DTexture), `higherOrderScatteringTexture`.

- 
A post-processing effect (`AerialPerspectiveEffect`) that renders atmospheric transparency and inscattered light, which can optionally render sun and sky irradiance as post-process lighting.

- 
Currently developed using GLSL. It does not use node-based TSL yet, and WebGPU is not supported, but both are planned.


**What `@takram/three-clouds` provides:**
- 
Geospatial volumetric clouds with features including Beer shadow maps (BSM) and shadows cast on scene objects, temporal upscaling and filtering, light shafts (crepuscular rays), and haze (sparse fog).

- 
Standard volumetric ray marching with scattering/absorption coefficients and dual-lobe Henyey-Greenstein phase function with configurable anisotropy parameters.

- 
Performs ray marching in the sun's orthographic projection and outputs the necessary values for computing the optical depth of the clouds (BSM) during the main camera's ray marching.

- 
Temporal upscaling reduces the number of texels to ray march in the clouds pass by 1/16.


**Key limitations:**
- 
Volumetric light shaft is not implemented in the atmosphere package as it requires ray tracing or an additional depth pass from the sun. You may notice scattered light is not occluded by scene objects.
 (Light shafts *are* available in the clouds package.)
- 
Suitable for large-scale scenes, but supports only Lambertian BRDF.

- 
Although you can generate custom precomputed textures, the implementation is effectively limited to Earth's atmosphere.


**Install:**
```bash
npm install @takram/three-atmosphere @takram/three-clouds postprocessing three
```

**R3F Usage (declarative):**
```tsx
import { EffectComposer } from '@react-three/postprocessing'
import { AerialPerspective, Atmosphere, Sky } from '@takram/three-atmosphere/r3f'
import { Clouds, CloudLayer } from '@takram/three-clouds/r3f'

const Scene = () => (
  <Atmosphere date={new Date()}>
    <Sky />
    <mesh><meshBasicMaterial /></mesh>
    <EffectComposer enableNormalPass>
      <Clouds disableDefaultLayers>
        <CloudLayer channel='r' altitude={750} height={650} />
        <CloudLayer channel='g' altitude={1000} height={1200} />
      </Clouds>
      <AerialPerspective sky sunLight skyLight />
    </EffectComposer>
  </Atmosphere>
)
```

**Vanilla Three.js Usage:**
```js
import { AerialPerspectiveEffect } from '@takram/three-atmosphere'
import { EffectComposer, RenderPass, EffectPass } from 'postprocessing'
import { HalfFloatType } from 'three'

const aerialPerspective = new AerialPerspectiveEffect(camera, {
  irradianceTexture,
  scatteringTexture,      // Data3DTexture
  transmittanceTexture,
  higherOrderScatteringTexture
})
const composer = new EffectComposer(renderer, { frameBufferType: HalfFloatType })
composer.addPass(new RenderPass(scene, camera))
composer.addPass(new EffectPass(camera, aerialPerspective, new ToneMappingEffect()))
```

## 2. Key GLSL Techniques Needed

### A. Precomputed Atmospheric Scattering (Bruneton Model)

The core principle, which UE5's Sky Atmosphere also uses, is precomputing LUTs:


The transmittance, scattering and irradiance textures are precomputed, and functions use them to compute the sky color, the aerial perspective, and the ground radiance. The single Rayleigh scattering, without its phase function term, plus the multiple scattering terms are stored in a `scattering_texture`.



The single Mie scattering is stored either separately in a `single_mie_scattering_texture`, or if the `COMBINED_SCATTERING_TEXTURES` preprocessor macro is defined, in the `scattering_texture` with Rayleigh and multiple scattering in RGB channels and the red component of single Mie scattering in the alpha channel.


**Key textures to precompute:**
1. **Transmittance LUT** (2D) — optical depth along a ray through the atmosphere
2. **Scattering LUT** (3D, `Data3DTexture`) — single + multiple scattering
3. **Irradiance LUT** (2D) — ground irradiance from sky
4. **Single Mie Scattering** (3D, optional separate texture)

### B. Volumetric Cloud Ray Marching

Based on techniques from Guerrilla Games (Horizon) and Frostbite (referenced by @takram/three-clouds):


To simulate anisotropic scattering in our cloud scene for each sampling point for a given light source, we can use a phase function. A common one is the Henyey-Greenstein phase function.


Key GLSL patterns:
```glsl
// Henyey-Greenstein phase function (dual-lobe)
float hg(float cosTheta, float g) {
    float gg = g * g;
    return (1.0 - gg) / (4.0 * PI * pow(1.0 + gg - 2.0*g*cosTheta, 1.5));
}
float dualLobePhase(float cosTheta, float g1, float g2, float w) {
    return mix(hg(cosTheta, g1), hg(cosTheta, g2), w);
}

// Beer-Lambert absorption
float transmittance = exp(-sigma_t * stepLength);

// Beer-Powder approximation (bright thin edges)
float beerPowder = exp(-density) * (1.0 - exp(-2.0 * density));
```


Blue noise dithering is leveraged to erase banding/layering effects due to lower step counts. This technique leverages a blue noise pattern, which has fewer patterns or clumps than other noises and is less visible to the human eye.


### C. Rayleigh + Mie Phase Functions

From `glsl-atmosphere` — the canonical minimal implementation:

```glsl
// Rayleigh phase
float pRlh = 3.0 / (16.0 * PI) * (1.0 + mu * mu);

// Mie phase (Henyey-Greenstein)
float pMie = 3.0 / (8.0 * PI) * ((1.0 - gg) * (mu*mu + 1.0))
           / (pow(1.0 + gg - 2.0*mu*g, 1.5) * (2.0 + gg));

// Optical depth per step (exponential height falloff)
float odStepRlh = exp(-height / rayleighScaleHeight) * stepSize;
float odStepMie = exp(-height / mieScaleHeight) * stepSize;
```

### D. God Rays (Crepuscular Rays)


The godrays effect works by raymarching through the scene and sampling the shadow map to determine which points are in shadow.


```js
import { GodraysPass } from 'three-good-godrays'
const godraysPass = new GodraysPass(directionalLight, camera, {
  density: 1/128, maxDensity: 0.5,
  edgeStrength: 2, edgeRadius: 2,
  distanceAttenuation: 2,
  color: new THREE.Color(0xffffff),
  raymarchSteps: 60, blur: true
})
composer.addPass(godraysPass)
```

### E. Height-Based Exponential Fog

For UE5-style Exponential Height Fog, Three.js only provides linear/exponential distance fog natively. For height-based fog, you need a custom shader:

```glsl
// UE5-style exponential height fog
uniform float fogDensity;
uniform float fogHeightFalloff;
uniform float fogStart;
uniform vec3 fogColor;
uniform vec3 fogInscatteringColor;
uniform vec3 sunDir;

float heightFog(vec3 worldPos, vec3 cameraPos) {
    float rayLength = length(worldPos - cameraPos);
    float avgHeight = (worldPos.y + cameraPos.y) * 0.5;
    float fogAmount = fogDensity * exp(-fogHeightFalloff * avgHeight);
    fogAmount *= (1.0 - exp(-fogHeightFalloff * rayLength))
               / fogHeightFalloff;
    return 1.0 - exp(-max(fogAmount, 0.0));
}
```


The @takram/three-clouds package also includes haze: an approximated haze effect that is inexpensive and recommended to keep enabled.

## 3. Performance Characteristics & GPU Cost

### `@takram/three-clouds` Performance


Volumetric clouds are not a lightweight effect. You might need to adjust the quality settings for your target devices. There are 4 quality presets: Low (disables shape detail, light shafts, and turbulence), Medium (disables light shafts and turbulence), and higher tiers.



Greater total cloud layer height increases computational cost, and excessive erosion reduces efficiency by causing rays to miss the clouds, leading to unnecessary sampling of the weather texture.



The number of cloud layers is limited to 4 because the coverage of all layers is packed into a texture and all layers are computed at once as vec4 in the shaders.


Key performance toggles:
- 
`lightShafts`: Whether to render light shafts (crepuscular rays) using additional ray marching. This enhances visual impact but is computationally expensive.

- 
`shapeDetail`: Whether to sample the shape detail texture. Enhances cloud details but is computationally expensive.

- 
Temporal upscaling reduces ray marched texels by 1/16. Recommended to keep enabled unless targeting very high-performance devices.

- 
Spatiotemporal blue noise (STBN) texture is used for stochastic sampling. Omitting it will make spatial and temporal aliasing highly noticeable.



If "Low" quality preset still does not meet your performance goal, then the author recommends considering a skybox instead.


### Budget Estimates for 60fps @ 1080p

| Layer | GPU Time (Mid-Range Desktop) | Technique |
|-------|-----|-----------|
| **Precomputed Sky** (Bruneton) | ~0.2–0.5ms | Single fullscreen quad, LUT lookups only |
| **Aerial Perspective** | ~0.3–0.8ms | Post-process, depth + LUT |
| **Volumetric Clouds** (Medium preset) | ~3–6ms | Ray marching, temporal upscale 1/16 |
| **Volumetric Clouds** (High preset) | ~6–12ms | + shape detail, light shafts, turbulence |
| **God Rays** | ~1–2ms | Screen-space, 60 march steps |
| **Height Fog** | ~0.1–0.3ms | Fullscreen quad, analytical |
| **Total Budget (Medium)** | ~5–9ms | Fits in 16.6ms frame |

### `three-good-godrays` Performance


When PCFShadowMap is used, Three.JS enables hardware shadow comparison mode on the depth texture, which is incompatible with the godrays shader. In this case, three-good-godrays automatically copies the shadow map depth data to a separate texture each frame, which adds a small performance overhead.

## 4. Three.js r184-Specific APIs


r184 (March 2026) memory cleanup: per-frame allocations were eliminated, helping complex scenes hold steady frame rates.



The r184 release improved memory management by eliminating per-frame object allocations. Previously, rendering 1,000 meshes at 60fps could generate 240,000 to 500,000+ unnecessary objects per second, overloading the garbage collector.


Key r184 features relevant to Vanguard1:

| Feature | Relevance |
|---------|-----------|
| 
**TSL (Three Shading Language)** — a JavaScript node-graph API introduced as stable in r184. You compose shading logic from nodes and assign to `material.colorNode` or `positionNode`.
 | Write-once shaders for WebGL+WebGPU |
| 
**TSL compiles to GLSL for WebGL and to WGSL for WebGPU.**
 | Future-proofing |
| 
**Volumetric clouds and atmosphere added to `webgl_loader_3dtiles` example**
 | Official example to reference |
| **`Data3DTexture`** (existing since WebGL2) | Required for Bruneton scattering LUTs |
| **`HalfFloatType` render targets** | HDR pipeline for atmosphere |
| 
**Reversed depth buffer support**
 | Better depth precision for large-scale scenes |
| **`WebGPURenderer` with WebGL2 fallback** | 
Since r171, WebGPU integration no longer requires bundler tweaks or polyfills, and silently falls back to WebGL 2.
 |

**Critical note for Vanguard1:** Since @takram/three-atmosphere 
is currently developed using GLSL and does not use node-based TSL yet
, it works with `WebGLRenderer`. For r184, stick with the WebGL path for the atmosphere stack while the WebGPU rewrite is ongoing. 
The author is actively working on WebGPU support, involving a complete rewrite of the API due to the transition from shader-chunk-based post-processing to a node-based approach, and it already performs much better.

## 5. Recommended Architecture for Vanguard1

```
┌───────────────────────────────────────────────────┐
│                 Render Pipeline                    │
├───────────────────────────────────────────────────┤
│ 1. Scene Pass (WebGLRenderer, MeshBasicMaterial)  │ ← Albedo only
│ 2. CloudsEffect (ray march, BSM, temporal upscale)│ ← @takram/three-clouds
│ 3. AerialPerspectiveEffect (Bruneton LUTs)        │ ← @takram/three-atmosphere
│    + inscatter + transmittance + sun/sky lighting  │
│ 4. GodraysPass (screen-space, shadow map sampling) │ ← three-good-godrays
│ 5. Height Fog (custom ShaderMaterial post-process) │ ← Custom GLSL
│ 6. ToneMappingEffect (ACES)                       │
│ 7. FXAA / TAA                                     │
└───────────────────────────────────────────────────┘
```

All using `pmndrs/postprocessing` EffectComposer (not Three.js built-in), with `HalfFloatType` frame buffers for HDR throughout.
