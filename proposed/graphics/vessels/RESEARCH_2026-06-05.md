### Key Loading Patterns

From Discover three.js: 
glTF files can contain models, animations, geometries, materials, lights, cameras, or even entire scenes, meaning you can create an entire scene in an external program then load it into three.js — and this entire scene fits in a single .glb file.



glTF is a specification for the efficient transmission and loading of 3D scenes and models. glTF minimizes both the size of 3D assets, and the runtime processing needed to unpack and use those assets.

## 3. Procedural Ship Hull & GLSL Shaders

### Vessel.js — Procedural Ship Hull Library (⭐ KEY FIND)


Vessel.js is a JavaScript library for conceptual ship design with an object-oriented paradigm. Vessel.js represents the vessel as an object, which is used to simulate different functionalities and behaviors.



Currently, the library includes methods for hydrostatic and stability calculations. The library is developed by Ship Design and Operation Lab at Norwegian University of Science and Technology (NTNU) in Ålesund.


- **GitHub**: `https://github.com/shiplab/vesseljs`
- **Live Examples**: `https://shiplab.github.io/vesseljs/`

Key examples from the Vessel.js wiki include:
- 
A 3D ship made from a ship specification, moving with waves in a regular wave ocean, with motion calculated with closed-form expressions.

- 
A fullscreen 3D visualization of an example ship specification (based on Ulstein PX121).

- 
The apps take a JSON file as input and plot a 3D image of the ship, with visualization made using three.js.


This is **directly applicable** to Vanguard1 — you can define vessels via JSON specifications and render procedural hulls in Three.js.

## 4. Instanced LOD Mesh for Vessels

### @three.ez/instanced-mesh (⭐ KEY FIND)


Enhanced InstancedMesh with frustum culling, fast raycasting (using a BVH), sorting, visibility, LOD, skinning and more.


Key features for Vanguard1:
- 
Shadow LOD: optimize shadow rendering with lower detail for distant instances. Skinning: apply skeletal animations to instances for more complex and dynamic movements.

- 
Per-instance frustum culling to skip rendering for out-of-view instances. Spatial indexing (dynamic BVH) to speed up raycasting and frustum culling. Sorting to reduce overdraw and manage transparent objects efficiently. Per-instance visibility to toggle visibility for each instance individually.


- **GitHub**: `https://github.com/agargaro/instanced-mesh`
- **npm**: `npm install @three.ez/instanced-mesh`

### LOD with 1 Million Instances Demo


The LOD implementation for InstancedMesh2 handles a million instances that change color and geometry depending on distance, done using a BVH.


- **Forum thread**: `https://discourse.threejs.org/t/instancedmesh-lod-1-million-instances/70748`

### Three.js Built-in LOD


The built-in LOD class adds a mesh that will display at a certain distance and greater — typically the further away the distance, the lower the detail on the mesh.


- **Docs**: `https://threejs.org/docs/pages/LOD.html`

### Three.js InstancedMesh


InstancedMesh is a special version of a mesh with instanced rendering support. Use this class to render a large number of objects with the same geometry and material(s) but with different world transformations.


- **Docs**: `https://threejs.org/docs/pages/InstancedMesh.html`

### Instancing Tutorial


With instancing, we can render hundreds of thousands, or even millions, of meshes with a single render call — but only if the meshes share the same geometry and material.


- **Tutorial**: `https://waelyasmina.net/articles/instanced-rendering-in-three-js/`

## 6. MeshStandardMaterial PBR for Ship Textures

### Core Properties for Ship Materials


roughness controls how rough the material appears: 0.0 means a smooth mirror reflection, 1.0 means fully diffuse. If roughnessMap is also provided, both values are multiplied. Default is 1. The green channel of the roughnessMap texture is used to alter the roughness.



The blue channel of the metalnessMap texture is used to alter the metalness of the material.



The normalMap texture creates a normal map. The RGB values affect the surface normal for each pixel fragment and change the way the color is lit. Normal maps do not change the actual shape of the surface.



Technical details of the approach used in three.js (and most other PBR systems) can be found in a paper from Disney, by Brent Burley.


### Recommended Ship Material Settings

For **steel hull** (naval grey):
- `metalness: 0.85`, `roughness: 0.45` — weathered metal
- Use a normal map for plate rivets and welding seams

For **painted superstructure**:
- `metalness: 0.1`, `roughness: 0.6` — painted surface


The material performance progression is: MeshBasicMaterial ➡ MeshLambertMaterial ➡ MeshPhongMaterial ➡ MeshStandardMaterial ➡ MeshPhysicalMaterial. The slower materials can make more realistic looking scenes but you might need to use faster materials on low powered or mobile machines.


- **Docs**: `https://threejs.org/docs/pages/MeshStandardMaterial.html`
- **Materials tutorial**: `https://threejs.org/manual/en/materials.html`
- **PBR tutorial**: `https://sbcode.net/threejs/meshstandardmaterial/`

## 7. Ship Silhouette / Outline Rendering

### OmarShehata/webgl-outlines (⭐ KEY FIND)


This renders outlines with a post-process shader that takes the depth buffer and a surface normal buffer as inputs, followed by an FXAA pass.



The technique uses a "normal buffer" in the outline pass to find inner edges. The source code is available on GitHub at https://github.com/OmarShehata/webgl-outlines.


The technique works by: 
computing the difference between the current pixel's depth value and its neighbors. A large depth difference indicates a distance gap (giving the outer boundary of an object).


- **GitHub**: `https://github.com/OmarShehata/webgl-outlines`
- **Tutorial**: `https://omar-shehata.medium.com/how-to-render-outlines-in-webgl-8253c14724f9`
- **Live demo**: Accepts drag-and-drop of .glb models

### Three.js Built-in OutlineEffect


An outline effect for toon shaders, usable only with WebGLRenderer. Usage: `const effect = new OutlineEffect(renderer); effect.render(scene, camera);` — imported from `'three/addons/effects/OutlineEffect.js'`.


- **Docs**: `https://threejs.org/docs/pages/OutlineEffect.html`

### Three.js OutlinePass (Post-processing)

For selectable object outlines (e.g., clicking on a vessel to highlight it), the `OutlinePass` is available in the post-processing pipeline. The key parameters include `edgeStrength`, `edgeGlow`, `edgeThickness`, `visibleEdgeColor`, and `hiddenEdgeColor`.

- **Forum discussion**: `https://discourse.threejs.org/t/how-to-render-full-outlines-as-a-post-process-tutorial/22674`

## 8. Animated Ship Propeller

### The Aviator Tutorial (Codrops)

The classic approach for animating rotating parts in Three.js is demonstrated in "The Aviator" tutorial, which shows propeller animation: 
The propeller rotation is achieved simply with `airplane.propeller.rotation.x += 0.3;` inside the animation loop, combined with smooth position interpolation for realistic movement.


- **Tutorial**: `https://tympanus.net/codrops/2016/04/26/the-aviator-animating-basic-3d-scene-threejs/`

### Implementation Pattern for Ship Propeller

For a GLTF model with named parts, you traverse the scene graph to find the propeller mesh, then rotate it each frame:

```javascript
// After loading GLTF
const gltf = await loader.loadAsync('models/ship.glb');
const propeller = gltf.scene.getObjectByName('Propeller');

// In animation loop
function animate() {
    propeller.rotation.z += speed * delta; // RPM-based rotation
    renderer.render(scene, camera);
    requestAnimationFrame(animate);
}
```
