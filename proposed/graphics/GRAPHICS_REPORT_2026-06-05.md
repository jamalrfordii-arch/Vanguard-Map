## Track 1 — Point Cloud & Terrain Rendering

**Two LOW-risk improvements that build on the existing splat shader without touching the four guarded uniforms.**

### Enhancement A: Normal-Map Hemisphere Lighting

Binds `terrain_normals.png` (already generated by `tools/generate_normals.py`) to the
splat cloud ShaderMaterial. In the fragment shader, after the existing brightness/gamma/saturation
pipeline, a hemisphere lighting term (cool sky fill on upward slopes, warm ground bounce on
downward slopes) plus a Lambertian key light are multiplied into the color. A `uNormalLightStrength`
uniform (default 0.35) controls the blend — setting it to 0 restores exact current appearance.

**What analysts see:** Mountain ranges develop natural slope shading — north faces go subtly
cool-dark, sun-facing slopes warm-bright. Valleys and bowls read as recessed. The topographic
depth already present in the point cloud gains a lighting layer that reinforces it.

**Safe rollback:** `window.splatCloud.material.uniforms.uNormalLightStrength.value = 0`

### Enhancement B: Biome-Aware Polar/Desert Tinting

Adds latitude and elevation as vertex varyings, then computes three independent biome masks
in the fragment shader: polar ice/tundra (>65° latitude), subtropical arid belt (16°–32°),
and high-altitude snowcap (>3000m). Each mask drives a partial desaturation + color-hue shift.
Controlled by `uBiomeStrength` (default 0.30).

**What analysts see:** Antarctica and the Arctic read as distinctly cool blue-white ice.
The Sahara/Arabian Peninsula push warm amber. Himalayan/Andean peaks acquire a gray-white
snowcap. Major biome bands emerge from the satellite color data without any new textures.

**Safe rollback:** `window.splatCloud.material.uniforms.uBiomeStrength.value = 0`

**Files:** `proposed/graphics/point-cloud/PROPOSALS_2026-06-05.md`, extracts in `enhancement_1.js` – `enhancement_6.js`

## Track 2 — Realism: Lighting, Atmosphere, Water

**Two MEDIUM-risk improvements. Both are gated behind sun-elevation checks so they degrade
gracefully at noon and are only dramatic at golden-hour/storm angles.**

### Enhancement A: God Rays in Existing Fog Pass

Patches `fogManager.js` to add screen-space radial volumetric rays (Mie scatter simulation)
that piggyback on the existing fog fragment work. The sun position is projected to screen UV
each frame; the fragment shader marches 12 samples radially toward that UV with exponential
decay. No new ShaderPass is added — the 12 samples add ~15% to the existing fog cost (~1–2 FPS).

**What analysts see:** At dawn/dusk, golden shafts fan outward from the sun position through
the fog layer. Mountains cast shadow corridors into the haze. The effect is absent at high
sun elevation and strong at horizon angles.

**Risk:** MEDIUM — modifies existing fog shader. Additive and sun-elevation gated, so worst case
is "fog unchanged."

### Enhancement B: Dynamic Foam & Spray on Ocean Surface

Patches `waterManager.js` to inject wave-breaking detection into the existing Gerstner wave
vertex/fragment shader. Where wave crest height exceeds a threshold (Jacobian-based breaking),
a foam texture alpha is blended in as white turbulence. Spray particles are emitted from
high-crest points using a shared `InstancedMesh` of 200 billboard quads.

**What analysts see:** Open ocean develops white-cap streaks on high waves. Ship wakes show
trailing foam turbulence. The ocean surface transitions from the current smooth animated
plane to one with weather-state-appropriate surface texture.

**Risk:** MEDIUM — modifies Gerstner wave GLSL. Wave parameters (waveA/B/C/D steepness,
wavelength) are not changed.

**Files:** `proposed/graphics/realism/PROPOSALS_2026-06-05.md`, extracts in `enhancement_1.glsl`

## Track 3 — Vessel & Aircraft Design

### Enhancement A: Multi-LOD Procedural Hull System

`vesselGeometryFactory.js` — replaces all `shapeBuilders` in `entityBuilder.js` with
`THREE.LOD` objects that serve three levels:

- **LOD0 (close, <30 scene units):** Full multi-mesh group with animated sub-objects — rotating
  radar dishes, engine exhaust cones, nav-light spheres, triplanar rust-streak shader on hulls.
- **LOD1 (mid, 30–120 units):** Simplified single merged `BufferGeometry` with baked vertex colors.
- **LOD2 (far, >120 units):** Single `Sprite` billboard — 1 draw call at cruising altitude.

**What analysts see:** Nearby vessels are visually distinct by class — a CARGO reads obviously
different from a TANKER or PATROL. Rust streaks, antifouling paint bands, and hull weathering
appear at close zoom. Distant vessels remain lightweight sprites.

**Performance:** Net draw-call *reduction* — LOD2 sprites replace 4-6 mesh groups per vessel.
Full code in `proposed/graphics/vessels/PROPOSALS_2026-06-05.md`.

### Enhancement B: Free GLTF Asset Pipeline

Documents the path to upgrading from procedural geometry to free CC0 GLTF models:

- **Kenney.nl** (`kenney.nl/assets`) — 30,000+ CC0 game assets; ship pack and vehicle pack
  include container ship, patrol boat, and aircraft in GLTF format.
- **FetchCFD** (`fetchcfd.com`) — cargo ship CFD-grade GLTF free for non-commercial use.
- **TurboSquid free tier** — 100+ free boat models, some GLTF.

The proposal includes a `GTLFVesselLoader` fallback wrapper that loads GLTF if present and
falls back to procedural geometry if not, keeping the codebase working during a staged migration.

**Risk:** LOW — additive, no existing code changed unless GLTF models are explicitly dropped in.

**Files:** `proposed/graphics/vessels/PROPOSALS_2026-06-05.md`

## Track 4 — Emerging Three.js & WebGL Techniques

### Technique A: BatchedMesh for Fleet Draw-Call Consolidation

Three.js `BatchedMesh` (stable since r155, full frustum-culling-per-item in r165) allows objects
with different geometries but the same material to be submitted as a single GPU draw call.

`vesselBatchManager.js` (full code in techniques proposals) pre-allocates one BatchedMesh per
material tier (hull, superstructure). `aisManager.js` calls `updateVessel(mmsi, pos, heading)`
instead of moving Groups. On integrated GPUs that are draw-call bound, expected 8–15 FPS gain
during 150+ vessel scenarios.

**Migration effort:** MEDIUM — vessel spawn/despawn/update lifecycle needs adaptation. Trail and
wake systems that reference vessel Groups need an indirection layer.

### Technique B: TSL Node Material for Terrain

Three.js Shading Language (TSL, r166+) compiles to GLSL for WebGL and WGSL for WebGPU from a
single JavaScript node graph. Migrating the splat cloud `ShaderMaterial` to `MeshBasicNodeMaterial`
with TSL nodes enables:

- Runtime effect toggling without shader recompilation
- Day/night/threat-overlay mode variants as `select(condition, a, b)` branches
- Future shader extensions that snap in as new nodes, no raw GLSL string editing

**Compatibility:** TSL with WebGLRenderer produces identical output to hand-written GLSL. The
auto-tuner's `uniforms.uBrightness.value = X` API works unchanged — TSL `uniform()` nodes are
plain JS objects with a `.value` property.

**Migration effort:** MEDIUM — requires rewriting both vertex and fragment shaders as TSL
node expressions. Payoff is easier future maintenance and WebGPU readiness.

**Files:** `proposed/graphics/techniques/PROPOSALS_2026-06-05.md`

1. Open the PROPOSALS file for the track you want
2. Find the enhancement
3. **For SHADER PATCHES:** locate the indicated section in the source file and replace
4. **For NEW MANAGERS:** copy the extracted `.js` file to the project root, wire into `main.js` following the Architecture Boundaries in `CLAUDE.md`
5. **Run the Visual Regression Guard before and after:**

6. If the guard fails, revert and adjust — do not tune uniforms to compensate
