## 1. IUU Fishing Intelligence & Dark Vessel Behavioral Analysis

**What is it?**
A real-time layer that fuses Global Fishing Watch's event and vessel-insight APIs with Vanguard1's existing AIS data to detect illegal, unreported, and unregulated (IUU) fishing — flagging AIS shutoff events, suspicious loitering, unauthorized fishing in marine protected areas, and at-sea transshipments between vessels.

**Why is it strategically valuable?**
IUU fishing is estimated at $23B annually and is tightly linked to organized crime, sanctions evasion, and forced labor. 
Global Fishing Watch provides insights showing whether a vessel is on an RFMO IUU vessel list, AIS coverage metrics, AIS off-events where a vessel has likely disabled transmission, fishing events in no-take marine protected areas, and fishing in areas with no known RFMO authorization.
 This transforms Vanguard1 from a vessel tracker into a maritime crime detection platform. 
GFW has released new research into behavior risk indicators of IUU fishing, offering a structured, data-driven approach to assessing vessel risk based on critical behavioral characteristics including who a vessel interacts with at sea and how it stays hidden without broadcasting positional data.


**Specific data sources:**
- **Global Fishing Watch Events API** (`https://globalfishingwatch.org/our-apis/documentation`): 
Events API provides encounters, loitering, port visits and fishing events based on AIS data.
 
The Insights API provides a set of vessel insights that bring together important information on a vessel's known activity and authorizations, supporting risk-based decision-making and operational planning.

- **GFW Datasets API**: 
Allows downloading fixed Infrastructure Detections from Sentinel-1 and Sentinel-2 in MVT format.

- **Free API registration** at `https://globalfishingwatch.org/our-apis/` — 
GFW has created a technology portal providing APIs that enable automatic connection of data and products to other systems so users can easily integrate datasets, code, and models.


**How it fits the existing architecture:**
Extends `aisManager.js` with a companion module. The Events API returns JSON with lat/lon coordinates for encounters, loitering events, and port visits — these map directly into `lonLatToScene()`. Vessels flagged as IUU risks would get a distinct ShaderMaterial glow (red pulsing ring). Encounter events (two vessels meeting at sea) would render as connecting arcs similar to `satArcManager`. Fishing effort heatmaps can be rendered as an instanced point cloud layer similar to `terrainBuilder`'s splat approach.

## 1. Ocean Current Flow Field

**Category:** surface
**Manager:** `OceanCurrentManager.js`
**Proposed file:** `proposed/ocean-current-flow/OceanCurrentManager.js`

### Strategic Value
Ocean surface currents are the invisible infrastructure of maritime operations — they dictate fuel-efficient routing, predict drift trajectories for SAR and oil spills, and explain anomalous vessel behavior. Overlaying animated current flow with live AIS positions instantly reveals which vessels are fighting currents on suspicious detours versus exploiting them for efficiency. This transforms the water surface from a static blue plane into a living operational environment, giving analysts intuitive understanding of the hydrodynamic context behind every vessel track.

### Data Source
```
NASA OSCAR 0.25° global surface currents via PO.DAAC Harmony API: https://harmony.earthdata.nasa.gov/C2102959417-POCLOUD/ogc-api-coverages/1.0.0/collections/all/coverage/rangeset?granuleId=G2753490775-POCLOUD&subset=lat(min:max)&subset=lon(min:max) — returns NetCDF with u/v velocity components updated every 5 days. Fallback: NOAA Global RTOFS GRIB2 at https://nomads.ncep.noaa.gov/pub/data/nccf/com/rtofs/prod/ for 72-hour forecast currents updated daily. Supplementary SST from Copernicus Marine Service CMEMS API at https://data.marine.copernicus.eu/ for particle coloring.
```

### Visual Concept
Tens of thousands of luminous particles streaming across the ocean surface, tracing the paths of real currents in real time. Gulf Stream appears as a bright fast-moving river of cyan particles; equatorial currents as broad amber bands; gyres as visible spiraling vortices. Particle brightness and length encode speed — fast boundary currents glow intensely with long tails, sluggish mid-ocean drift shows as dim short dots. Particles fade in at random positions and fade out after a configurable lifespan, creating the organic flowing-smoke aesthetic of earth.nullschool.net but in full 3D with perspective. When vessels are visible, current arrows subtly appear beneath them showing the local flow vector. The layer sits just above the Gerstner wave surface at Y≈0.05, compositing naturally under fog and TAA.

### Implementation Approach
GPGPU ping-pong particle advection using two WebGLRenderTargets as Frame Buffer Objects. Parse OSCAR NetCDF u/v grids into a pair of DataTextures (1440×720 for 0.25° global, RGBA Float32, R=u G=v). Advection shader samples the velocity texture at each particle's XZ position (mapped to UV), applies dt-scaled displacement, writes new position to output FBO. Particles that exit bounds or exceed max age get respawned at random ocean positions using a blue-noise seed texture. Render pass reads the position FBO as a texture attribute on a Points mesh with custom ShaderMaterial — vertex shader reads particle XZ from FBO texel, maps through lonLatToScene(), fragment shader draws soft billboard sprites with alpha falloff. Trail effect via a third FBO that accumulates previous positions with exponential decay, rendered as GL_LINES between current and previous position. Particle color sampled from an optional SST DataTexture mapped to a thermal ramp uniform. Layer registered with layerManager under 'surface' group. Dispatch 'vg1:ocean-current:loaded' on data parse complete. Config constants: OCEAN_CURRENT_PARTICLE_COUNT (65536), OCEAN_CURRENT_SPEED_SCALE (0.0003), OCEAN_CURRENT_PARTICLE_LIFETIME (200 frames), OCEAN_CURRENT_FADE_OPACITY (0.92), OCEAN_CURRENT_TEXTURE_SIZE (256 — the sqrt of particle count for square FBO).

### Research Notes
OSCAR provides 0.25° near-global coverage (80°N-80°S) of ocean surface velocity derived from satellite altimetry, scatterometry, and SST. Data is freely available via NASA Earthdata with no API key for public collections — Harmony API supports spatial/temporal subsetting and format conversion. NetCDF-js (https://github.com/cheminfo/netcdfjs) parses NetCDF in browser without server dependencies. The GPGPU particle technique is well-proven in Three.js — the canonical approach uses FBOs where each pixel stores one particle's XYZ as RGB channels, enabling massively parallel advection on GPU. earth.nullschool.net demonstrates the visual power of this approach in 2D; MapboxGL's globe view has implemented similar 3D particle flows. Key implementation reference: Three.js GPGPU birds example (three/examples/webgl_gpgpu_birds) demonstrates the ping-pong FBO pattern. For production, particle count of 65536 (256×256 FBO) provides dense visual coverage while staying within mobile GPU budgets. RTOFS provides forecast data enabling a '72-hour prediction' scrubber. Copernicus Marine CMEMS requires free registration and provides higher-resolution regional products if needed.

## 2. Satellite RF Interference Heatmap

**Category:** atmosphere
**Manager:** `RfiHeatmapManager.js`
**Proposed file:** `proposed/rf-interference-heatmap/RfiHeatmapManager.js`

### Strategic Value
Space-based RF interference detection is effectively open-source ELINT derived from NASA science data. SMAP's L-band radiometer reveals undeclared radar emitters, GPS/GNSS jammers, and active electronic warfare zones worldwide — the Ukraine conflict produced massive observable RFI signatures from drone countermeasures visible from orbit. Correlating persistent RFI hotspots with military installations, conflict boundaries, shipping lanes, and the existing gpsJammingManager ground-truth data creates a multi-source electronic order of battle that no single sensor provides alone. This gives analysts a dimension of intelligence — the electromagnetic environment — that is otherwise invisible on a map.

### Data Source
```
NASA SMAP L1B/L3 RFI data via NSIDC DAAC: https://nsidc.org/data/spl1btb/versions/3 (L1B brightness temperatures with RFI flags) and pre-processed RFI maps at https://smap.jpl.nasa.gov/rfi/ providing 0.25°×0.25° gridded RFI persistence as percentage over 4-week windows. Programmatic access via NASA CMR API: https://cmr.earthdata.nasa.gov/search/granules.json?collection_concept_id=C2531308461-NSIDC_ECS&temporal=2024-01-01T00:00:00Z,2024-12-31T23:59:59Z — returns granule download links. NASA Earthdata bearer token auth (free registration at https://urs.earthdata.nasa.gov/). For near-real-time complementary data: Spire RF tasking API at https://documentation.spire.com/ provides targeted RF collections in IQ/JSON/CSV format (commercial, requires contract).
```

### Visual Concept
A translucent heatmap overlay draped across the globe's surface showing RF interference intensity. Clean spectrum appears as fully transparent; low RFI as cool blue shimmer; moderate interference as amber glow; intense persistent RFI as angry red-orange hotspots that pulse slowly. The layer sits at Y≈0.08, just above water and terrain surfaces, with depth-based opacity falloff so it fades when the camera is far away and intensifies on zoom. Known military installations and conflict zones light up as bright emission clusters. A temporal scrubber allows analysts to animate RFI persistence week-by-week, revealing the electromagnetic signature of conflict escalation — Ukraine, Middle East, South China Sea hotspots visibly intensify over time. When combined with gpsJammingManager's ground-reported jamming circles, analysts see both the cause (RFI emitters from space) and effect (GPS disruption on the ground) simultaneously.

### Implementation Approach
Pre-process SMAP 0.25° RFI persistence grids into a 1440×720 single-channel Float32 DataTexture (one value per grid cell: RFI persistence percentage 0-100). Create a PlaneGeometry(worldWidth, worldHeight, 1, 1) positioned at Y=0.08 with custom ShaderMaterial. Vertex shader is passthrough; fragment shader samples the RFI DataTexture at UV derived from world XZ position via sceneToLonLat(), applies a configurable color ramp uniform (vec3 array: transparent→blue→amber→red), and outputs with alpha proportional to RFI intensity multiplied by a distance-based fade uniform updated from camera position. Add slow pulsing animation for cells above a HIGH_RFI_THRESHOLD via sin(time) modulating emission. For temporal animation, load multiple monthly DataTextures into a texture array and lerp between adjacent months based on a timeline uniform driven by UI scrubber. Dispatch 'vg1:rfi-heatmap:month-changed' when temporal index updates so gpsJammingManager can synchronize its display period. Mask land vs ocean using the existing terrain/water boundary data to avoid rendering over terrain where terrainBuilder already provides surface detail — or render over both with additive blending (THREE.AdditiveBlending) so it composites as a glow. Register with layerManager under 'atmosphere' group. Config constants: RFI_HEATMAP_OPACITY_MAX (0.6), RFI_HEATMAP_HIGH_THRESHOLD (40.0 — persistence %), RFI_HEATMAP_PULSE_SPEED (0.8), RFI_HEATMAP_COLOR_RAMP ([0x000000, 0x1a3a5c, 0xd48a2e, 0xff2200]), RFI_HEATMAP_ALTITUDE (0.08).

### Research Notes
SMAP (Soil Moisture Active Passive) launched 2015, operates at 1.4 GHz L-band which is a protected radio astronomy band — any signal detected here is by definition interference, making RFI extraction unambiguous. JPL publishes pre-computed RFI maps at https://smap.jpl.nasa.gov/rfi/ showing global persistence grids on 0.25° resolution, updated regularly. The Ukraine conflict produced dramatic observable RFI increases visible in SMAP data as both sides deployed drone jammers and EW systems operating near L-band. SMAP RFI is measured as the delta in Kelvin between pre- and post-RFI-filtered antenna temperatures; persistence is the percentage of measurements flagged over a 4-week window. Data is distributed as HDF5 files via NSIDC — h5wasm (https://github.com/usnistgov/h5wasm) enables browser-side HDF5 parsing, or pre-process server-side to PNG/binary. The 1440×720 grid at Float32 is only ~4MB per monthly snapshot, making client-side multi-month animation feasible with <50MB total. This complements gpsJammingManager (which shows receiver-reported GNSS disruption from gpsjam.org) by providing the emitter-side view from space — the two layers together create a complete EW picture. For higher-fidelity commercial RF data, Spire's constellation provides targeted RF collection but requires commercial agreement. CelesTrak also tracks some known jamming-capable satellite payloads that could be cross-referenced.
